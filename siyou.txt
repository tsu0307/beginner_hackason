アプリ概要

AIと対話しながら人生の分岐を現実的にシミュレーションし、時間軸付きで可視化できる「人生 if アプリ」

本アプリは、ユーザーが自分の人生における出来事や現在の状況を入力すると、AIがその先に起こりうる2つの分岐を生成し、ユーザーが選択を重ねながら人生の可能性を体験できるアプリである。
単に理想的な未来を提示するのではなく、迷い、負担、人間関係、能力差、金銭面など、現実の人生に起こりうる要素も踏まえた分岐を提示することで、より自然で納得感のあるシミュレーションを目指している。

各分岐では、次に起こりうる選択イベントに加え、「安定度」「挑戦度」を表示する。ユーザーがその中から1つを選択すると、AIがその選択の先に起こりうる結果の概要と「幸福度」を生成し、その結果をもとにさらに次の2つの分岐を提示する。この流れを繰り返すことで、ユーザーは自分の人生の可能性を段階的かつ継続的に体験できる。

また、AIが提案する分岐に加えて、ユーザー自身が新たな分岐を追加することも可能である。これにより、AI任せではなく、ユーザーの想像や意思も反映した柔軟な人生シミュレーションが実現できる。

さらに、各ノードには西暦と年齢を表示する。これは単純に分岐数で年齢を増やすのではなく、各イベントに設定された duration をもとにアプリ側で計算することで、同一年内の出来事では年齢を進めず、進学・浪人・就職などの節目だけ自然に反映できる仕組みとしている。これにより、人生の時間軸を違和感なく把握できる。

加えて、これまでの選択履歴と各時点での結果・幸福度をもとに、人生ルート全体を1本のストーリーとして要約できるため、ユーザーは分岐の連続を「自分の人生の流れ」として振り返ることができる。

機能一覧
1. 人生イベント入力機能

ユーザーが現在の状況や人生の出来事を入力する機能。
進学、就職、転職、留学、起業、人間関係の変化など、さまざまなライフイベントを起点として分岐生成を開始できる。

2. AIによる2分岐生成機能

入力された状況と、それまでの人生ストーリーをもとに、AIが次に起こりうる2つの分岐を生成する機能。
2つの分岐は、方向性が明確に異なる内容となるよう設計しており、片方を比較的「安定寄り」、もう片方を比較的「挑戦寄り」とすることで、ユーザーが選択肢を比較しやすい形にしている。

各分岐では、以下の情報を表示する。

選択イベント

安定度（高・中・低）

挑戦度（高・中・低）

duration（その出来事までの経過年数）

3. 分岐選択機能

提示された2つの分岐のうち、ユーザーが1つを選んで次のステップへ進める機能。
分岐をただ眺めるだけでなく、選択を重ねることで体験型のシミュレーションとして成立させている。

4. 選択結果表示機能

ユーザーが選択した分岐に対して、AIがその後に起こりうる結果の概要を生成する機能。
結果は成功だけを描くのではなく、迷い、不安、停滞、負担なども含めて現実的に表現する。あわせて「幸福度」を高・中・低で表示し、その時点の心理的な充実度や満足感の傾向を示す。

表示項目は以下の通り。

結果概要

幸福度（高・中・低）

duration

5. 次の分岐再生成機能

選択後の結果をもとに、AIがさらに次の2分岐を生成する機能。
「分岐生成 → 選択 → 結果表示 → 次の分岐生成」という流れを繰り返すことで、人生の可能性を連続的に体験できる。

6. ユーザー分岐追加機能

AIが提案した2つの分岐に加え、ユーザー自身も新しい分岐案を自由入力で追加できる機能。
入力された分岐はAIが整形し、他の分岐と同じ形式に揃えることで、アプリ全体の表示や管理と整合性を保つ。

7. 樹形図表示機能

人生ルート全体をノードと線で可視化する機能。
分岐の連続を樹形図として表示することで、どの選択からどの未来に進んだのかを直感的に把握できる。横スクロール型のUIを採用することで、複数の分岐が増えても見渡しやすい構成としている。

8. 西暦・年齢自動計算表示機能

各ノードに西暦と年齢を表示する機能。
年齢や西暦は分岐の深さではなく、各イベントに設定された duration をもとにアプリ側で計算する。これにより、同一年内の出来事では年齢が進まず、浪人・進学・就職・転職など、年単位の変化がある場面だけ自然に反映される。
人生ツリーの時間軸を自然に保ち、不自然な年齢表示や西暦のずれを防ぐ重要な機能である。

9. 人生ストーリー要約機能

これまでの選択履歴と各時点での結果、幸福度の流れをもとに、人生ルート全体を短い文章で要約する機能。
単なる履歴の羅列ではなく、「充実していた時期」「迷いが大きかった時期」「安定していた時期」などを自然な物語としてまとめることで、ユーザーが自分の人生の流れとして読み返せるようにしている。

10. ユーザー属性登録機能

年齢、価値観、興味、性格傾向などの属性を登録する機能。
これらの情報は、AIによる分岐生成、結果生成、ストーリー要約に活用され、よりユーザーに合った自然なシミュレーションにつながる。

11. プロンプト制御機能

用途ごとに専用のAIプロンプトを使い分ける機能。
具体的には、以下の4種類のプロンプトを用意している。

2分岐生成用プロンプト

選択結果表示用プロンプト

人生ストーリー要約用プロンプト

ユーザー追加分岐整形用プロンプト

これにより、各場面で必要な情報だけを安定してJSON形式で取得し、アプリ内で扱いやすい構造にしている。

1. AIによる2分岐生成用プロンプト
あなたは、ユーザーの人生の選択肢を現実的にシミュレーションするアシスタントです。
理想化しすぎず、現実の人生にある迷い、負担、環境変化、能力差、人間関係、金銭面なども踏まえて分岐を提案してください。

ユーザーの現在の状況をもとに、次に起こりうる人生の分岐を2つ生成してください。

【目的】
- 2つの分岐は明確に異なる方向性にすること
- 片方は比較的「安定寄り」、もう片方は比較的「挑戦寄り」にすること
- 内容が似すぎないようにすること
- どちらの分岐も美化しすぎないこと
- 成功を前提にしすぎず、現実的な可能性として提案すること
- 各分岐は、次に選ぶ行動や出来事として自然な内容にすること
- 各分岐には、その出来事までに経過する年数 duration を含めること

【出力ルール】
- 分岐は2つだけ出力する
- 分岐生成時に表示する情報だけを出力する
- 表示項目は以下に限定する
  - event
  - stability
  - challenge
  - duration

【評価基準】
- stability と challenge は「高・中・低」で出力する
- duration は 0 以上の整数で出力する
- 同一年内の出来事は duration = 0 とする
- 浪人、留年、進学、就職、転職など年単位の変化が自然な場合のみ duration を 1 以上にする
- 分岐イベントは短くわかりやすい表現にする
- 2つの分岐は方向性が明確に異なるようにする

【ユーザー情報】
- 年齢: {age}
- 価値観: {values}
- 興味: {interests}
- 性格傾向: {personality}
- 現在の出来事・状況: {current_event}
- これまでの人生ストーリー要約: {story_summary}

【出力形式】
JSONのみで出力してください。説明文や前置きは不要です。

{
  "branches": [
    {
      "event": "",
      "stability": "高",
      "challenge": "低",
      "duration": 0
    },
    {
      "event": "",
      "stability": "低",
      "challenge": "高",
      "duration": 1
    }
  ]
}

2.選択後の結果表示用プロンプト
あなたは、ユーザーが選んだ人生の分岐の結果を、現実的かつ簡潔に要約するアシスタントです。
理想化しすぎず、良い面だけでなく負担や不確実性も含めて、自然な人生の流れとして表現してください。

ユーザーが選択した分岐に対して、その後に起こりうる結果の概要を生成してください。

【目的】
- ユーザーが選んだ分岐の結果を、短い文章でわかりやすく示すこと
- 成功だけでなく、迷い、負担、停滞、不安定さなども必要に応じて含めること
- 必ず幸せな結果にしないこと
- 未来を断定せず、「こうなる可能性がある」という自然な表現にすること
- 結果には幸福度をつけること
- 次の分岐生成につながるような、人生の中間結果として自然な内容にすること
- 結果に至るまでの経過年数 duration を含めること

【出力ルール】
- 出力する項目は以下に限定する
  - result_summary
  - happiness
  - duration

【評価基準】
- result_summary は2〜4文程度で書く
- happiness は「高・中・低」で出力する
- duration は 0 以上の整数で出力する
- 同一年内の変化なら duration = 0 とする
- 年単位の時間経過が自然な場合のみ duration を 1 以上にする
- 結果は希望を持たせすぎず、悲観に寄せすぎず、現実的にする
- ユーザーの価値観や性格に照らして、その人らしい反応や変化を含めてもよい

【ユーザー情報】
- 年齢: {age}
- 価値観: {values}
- 興味: {interests}
- 性格傾向: {personality}
- これまでの人生ストーリー要約: {story_summary}
- 今回ユーザーが選んだ分岐: {selected_branch}

【出力形式】
JSONのみで出力してください。説明文や前置きは不要です。

{
  "result_summary": "",
  "happiness": "中",
  "duration": 0
}

3.人生ストーリー要約用プロンプト
あなたは、ユーザーがこれまで選んできた人生ルートを、1本の人生ストーリーとして簡潔に要約するアシスタントです。

【目的】
- これまでの選択の流れを自然な文章で振り返れるようにすること
- 時系列がわかるように整理すること
- 良い出来事だけでなく、迷いや変化も含めてまとめること
- 各選択後の幸福度の変化も踏まえて、人生全体の流れが伝わるようにすること
- ユーザー本人が「自分の人生の流れ」として読める文章にすること

【出力ルール】
- 150〜250文字程度で要約する
- 読みやすい自然な日本語にする
- 選択の連続がわかるようにする
- 必要に応じて転機や迷いも含める
- 幸福度はそのまま並べるのではなく、「充実していた時期」「迷いが大きかった時期」「安定していた時期」など、流れとして文章に反映する
- 幸福度が高い選択ばかりでも単純に美化せず、背景にある努力や迷いがあれば含める
- 幸福度が低い時期がある場合は、人生の転機や停滞として自然に要約へ反映する

【入力情報】
- ユーザー属性:
  - 年齢: {age}
  - 価値観: {values}
  - 興味: {interests}
  - 性格傾向: {personality}
- これまでの選択履歴: {route_history}
- 各選択後の結果一覧: {result_history}

【出力形式】
JSONのみで出力してください。説明文や前置きは不要です。

{
  "story_summary": ""
}

- 年単位の変化が自然な場合のみ duration を 1 以上にする
